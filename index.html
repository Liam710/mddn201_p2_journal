A bit about my design choices
and the thought behind them

Aesthetic

I kept it simple for this prototype. I followed the design concepts I developed, but I cut out some of the complexity to create a much more streamlined experience with less visual fluff. I cut the colour palette down to simply dark grey, white and magenta. This plays into the more conventional modern "mobile" design many sites are going for these days. To make my prototype easy to understand and to approach, I felt it necessary to emulate this style, as my target demographic is mainly those who aren't tech savvy.

Interface

Again, mostly drawing from the psuedo-mobile UX that most modern websites strive for. I tried to use text sparingly, to remove ambiguity and add to the streamlined experience. I used iconographic arrows on tabs to make it obvious where to click, as well as unifying all clickable objects with the same colour. This convention, I feel, helps direct the user to interactive elements.

For the pop up articles, I used an overlay. This paired with a transparent grey background, directs focus in such a way that it doesn't ever separate the user from the page, yet completely distiguishes the pop-up from the main page. This is another choice I made to create an effortless user flow, as well as keeping it intuitive.

Recieving the same information regardless of the answer was a byproduct of an error I made when linking the pages together. I realised that it would be valuable to get the same information when getting a question right, as it may have been an accident, or assumed correct for the wrong reason. It gets the same amount of info out to every user in a digestable and understandable fashion.

Reflection

I learned a lot about prototyping a web-based project. It made me think about the user experience more than I normally would, as opposed to focussing on aesthetic. While both are important, I feel like in the past I undervalued the UX aspect. Overall, I am reasonably happy with my prototype, and I think it communicates what I was aiming for when I began this project. If I were to do it again, I would probably use a different prototyping tool, as I found it difficult to create anything particularly compelling with Adobe XD.
